## Purpose of Redesign

This redesign proposes a modernised 12-week schedule for ISYS2001 that foregrounds computational thinking, AI fluency, and low-code toolchains while meeting all existing **Unit Learning Outcomes (ULOs)** and syllabus requirements. Syntax memorisation is deprioritised in favour of planning, logic, and problem decomposition using AI and visual tools.

### Unit Summary (as it stands)
The syllabus and ULOs emphasize:

- Basic programming using Python
- IDE use and debugging
- Data types, control structures, and algorithm
- Testing strategies and optimisation
- Real-world problem-solving with social/ethical awareness
- Project-based, AI-enhanced learning (already embedded)

## Assessment Overview

### 1. **Low-Code Diaries** (30%)

**Description:** Weekly reflections documenting project work, logic, and AI/tool interactions.
**Submission:** At least 8 of 9 weekly entries required for full marks
**Format:** Structured template with prompts and optional screenshots/code.
**Assessment Type:** Pass/fail per entry, scaled by completion

**Rubric Highlights:**

* Effort and engagement
* Logic explanation
* AI/tool evaluation
* Troubleshooting insight
* Clear and structured presentation

### 2. **Programming Project: "Capstone App Build"** (30%)

**Description:** Design and build a real-world solution using low-code or minimal-code tools.
**Tooling Options:** App Inventor, Replit (AI IDE), Thonny (optional Python), n8n
**Deliverables:** Functional app, documentation, screenshots, GitHub repo or shared folder
**Assessment Criteria:** Planning, functionality, testing, clarity, and creativity

### 3. **Applied Programming Portfolio: "Explain, Adapt, Build"** (40%)

**Format:** Open-book take-home assessment (exam period)
**Includes:**

* AI-refinement tasks
* Debugging prompts
* Code modification
* Short function writing and rationale
* Ethical reflection on AI and software design
